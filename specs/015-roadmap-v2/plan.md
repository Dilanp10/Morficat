# Plan: Roadmap v2+

**Status**: Backlog

## No es un plan de implementación

Esta spec sirve como referencia para futuras specs. Cada feature de la lista necesitará su propia carpeta `specs/NNN-feature/`.

## Criterios para promover una feature del backlog

Antes de iniciar una spec de v2:

1. ¿El MVP está realmente cerrado? (Spec 014 completa)
2. ¿Hay validación de que los usuarios la piden?
3. ¿La feature es coherente con el dolor protagonista (descubrimiento gastronómico)?
4. ¿Hay capacidad de mantenerla? (No agregar features que requieran soporte 24/7)
